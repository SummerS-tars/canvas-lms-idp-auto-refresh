### **复旦 elearning 平台自动化工具说明文档**

#### **1. 平台背景介绍**
复旦大学的 **elearning 平台**底层采用了国际主流的开源学习管理系统 **Canvas LMS**。

- **认证体系：** 该平台深度集成了复旦大学的**统一身份认证系统 (UIS/IDP)**。用户在访问时，需要先通过 UIS 进行身份验证，获取 JWT 票据后，再跳转回 Canvas 域换取会话 Cookie。
- **API 机制：** Canvas 本身提供了功能强大的 REST API，允许开发者通过生成“访问许可证 (Access Token)”来读取课程信息、下载作业或提交数据。

#### **2. 脚本的核心作用**
在实际使用中，由于学校安全策略的限制，手动生成的 API Token 往往会因为 SSO 会话过期（通常为 24 小时左右）而失效，导致自动化服务中断。
本脚本（`elearning_login.py` 等组件）的作用是实现**全自动化的身份闭环**：

1. **模拟登录：** 自动处理 UIS 登录过程，包括动态获取 RSA 公钥并对密码进行非对称加密。
2. **跨域授权：** 完成从 UIS 到 Canvas 的票据交换，获取合法的 `_normandy_session` 等关键 Cookie。
3. **Token 轮换：** 借助已获取的会话，直接调用 Canvas API 签发新的访问许可证（Access Token），从而规避手动续期的麻烦。

#### **3. 典型应用场景**

- **AI Agent（如 OpenClaw）持续访问：** 通过将此脚本作为前置服务，AI Agent 可以保持 24/7 不间断地访问 elearning。即使原始 Token 失效，Agent 也能自动触发此脚本重新获取权限。
- **自动化数据采集：** 用于开发个人课程助手，自动同步课程表、截止日期（Deadlines）或未读通知到日历和提醒事项中。
- **集成协作工具：** 例如将 Canvas 的更新实时推送到微信、钉钉或游戏服务器（如 Minecraft 聊天桥）中。

#### **4. 技术实现逻辑简述**
脚本采用了**轻量级的请求模拟流（Requests-based Flow）**：

- **无须浏览器：** 相比 Selenium 等重型工具，它不依赖浏览器内核，对服务器资源占用极低，非常适合部署在 Linux 云服务器上。
- **安全合规：** 严格遵循复旦 UIS 的 RSA 加密规范，确保账号密码在传输过程中的安全性。
